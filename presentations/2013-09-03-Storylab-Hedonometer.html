nginx (a reverse proxy) listens on port 80, and responds to requests based on a configuration that looks like:


# /etc/nginx/sites-available/www

server {
    listen 80;
    server_name hedonometer.org;
    access_log /var/log/nginx/sample_project.access.log;
    error_log /var/log/nginx/sample_project.error.log;

    add_header 'Access-Control-Allow-Origin' 'http://www.uvm.edu';

    rewrite  ^/(\?.*)?$  /index.html$1  permanent;

    location /static { # STATIC_URL
    	autoindex on;
        alias /usr/share/nginx/wiki/mysite/mysite/static/; # STATIC_ROOT
        expires 30d;
    }

    location /media/ { # MEDIA_URL
        alias /usr/share/nginx/wiki/mysite/media; # MEDIA_ROOT
        expires 30d;
    }

    location / {
        include fastcgi_params;
        fastcgi_pass 127.0.0.1:8080;
	fastcgi_param SCRIPT_NAME "";
    }
}

  1. load data
  2. make a selection
  3. apply data
  4. style with data
  5. add actions
